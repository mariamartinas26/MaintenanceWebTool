Specificații Software – Maintenance Web Tool

Proiect realizat de: Martinas Ioana Maria si Sacarescu Rebecca Maria

Rezumat

Acest document prezintă specificațiile aplicației Maintenance Web Tool – un sistem informatic pentru gestionarea programărilor și operațiunilor într-un service de motociclete, biciclete și trotinete. Aplicația permite clienților să efectueze programări, iar personalului administrativ să gestioneze eficient reparațiile, stocurile și relațiile cu furnizorii.

1. Introducere

Maintenance Web Tool este o aplicație web destinată managementului digital al activităților într-un service. Clienții pot face programări, iar personalul tehnic și administrativ beneficiază de funcționalități specifice rolurilor atribuite de manager. Toate conturile necesită aprobare.

2. Scopul sistemului

Scopul aplicației este să digitalizeze complet procesul de programare, gestiune a stocurilor și relaționare cu furnizorii în cadrul unui service. Se urmărește optimizarea fluxului de muncă, creșterea transparenței și reducerea erorilor umane.

3. Descrierea generală

  • Client: creează programări și atașează fișiere media
  • Manager: aprobă utilizatori și le atribuie roluri
  • Admin: aprobă/respinge programări, gestionează stocuri, face comenzi
  • Contabil: gestionează furnizori, import/export date

4. Cerințe funcționale

  1. Sistemul permite înregistrarea utilizatorilor și aprobarea de către manager
  2. Clientul poate consulta calendarul și face o programare
  3. Formularul de programare permite atașarea de imagini și clipuri video
  4. Adminul poate accepta sau respinge o cerere, cu justificare
  5. Adminul poate vizualiza și gestiona stocurile de piese
  6. Adminul poate crea comenzi de piese către furnizori
  7. Contabilul poate importa date în format CSV și JSON și le poate exporta inclusiv în PDF

5. Cerințe nefuncționale

  • Aplicația trebuie să răspundă în maximum 2 secunde la orice cerere validă
  • Autentificarea și autorizarea sunt realizate folosind JWT
  • Persistența datelor este asigurată de o bază de date PostgreSQL
  • Aplicația trebuie să fie accesibilă de pe desktop și mobil
  • Datele trebuie stocate criptat și în siguranță

6. Arhitectura sistemului

Sistemul este dezvoltat folosind HTML, CSS, JavaScript pur pe partea de client, iar pe backend Node.js. Datele sunt gestionate printr-o bază de date PostgreSQL. Autentificarea se face prin JWT.

7. Funcționalitate Bonus – Simulare activitate

Sistemul permite simularea activității service-ului pe intervale de timp (luni, ani), oferind rapoarte și statistici privind volumul programărilor, piesele folosite, comenzile către furnizori și gradul de ocupare.

8. Concluzii

Maintenance Web Tool oferă o soluție digitală completă pentru gestionarea modernă a unui service de reparații. Este scalabilă, sigură și intuitivă, susținând toate rolurile implicate în activitatea zilnică.
